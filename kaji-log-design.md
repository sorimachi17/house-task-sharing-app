# 家事ログ(KAJI LOG)設計書

Claude Codeへの実装指示書。この文書だけで実装が完結するように書いてある。

---

## 1. 概要

同棲カップル2人のための家事記録・可視化アプリ。

- 目的: **記録と見える化**。誰が・何を・いつ・どのくらいやったかを共有する
- ポイント制や評価機能は不要。回数と時間帯が見えればよい
- スマホブラウザで使う前提(モバイルファースト)
- ホスティング: **GitHub Pages**(静的ファイルのみ)
- データ共有: **Supabase**(無料枠、フロントから直接APIを叩く。自前サーバーなし)
- ビルド不要構成: バニラJS + Supabase JS(CDN読み込み)。`index.html` / `style.css` / `app.js` / `config.js` の4ファイル構成

## 2. ユーザーモデル

- ユーザーは固定2人。ログイン機能なし
- `config.js` に表示名と色を定義し、画面上部のトグルで「いま操作しているのは誰か」を切り替える(選択は localStorage に保存し、次回以降デフォルトにする)
- 実施者は3値: `a` / `b` / `both`(二人でやった)
- **`both` の記録は、集計時にA・B両方に1件ずつカウントする**

```js
// config.js
const CONFIG = {
  SUPABASE_URL: "https://xxxx.supabase.co",
  SUPABASE_ANON_KEY: "eyJ...",
  USERS: {
    a: { name: "マサ", color: "#2A9D8F" },   // 表示名は利用者が書き換える
    b: { name: "○○", color: "#E9A03B" },
  },
};
```

## 3. データモデル(Supabase / PostgreSQL)

```sql
-- 家事マスタ
create table chores (
  id uuid primary key default gen_random_uuid(),
  name text not null,
  category text not null,
  sort_order int not null default 0,
  is_active boolean not null default true,
  created_at timestamptz not null default now()
);

-- 実施記録
create table logs (
  id uuid primary key default gen_random_uuid(),
  chore_id uuid not null references chores(id),
  done_by text not null check (done_by in ('a', 'b', 'both')),
  done_at timestamptz not null default now(),
  created_at timestamptz not null default now()
);

create index idx_logs_done_at on logs (done_at);

-- RLS: 匿名キーでフルアクセス(2人用プライベートアプリのため)
alter table chores enable row level security;
alter table logs enable row level security;
create policy "anon all chores" on chores for all to anon using (true) with check (true);
create policy "anon all logs" on logs for all to anon using (true) with check (true);
```

## 4. 画面仕様

下部固定のタブバーで4画面を切り替える。全画面共通で、ヘッダーに「操作ユーザー切り替えトグル(A/B)」を置く。

### 4.1 記録タブ(ホーム)

- カテゴリごとにグループ化した家事リスト。1行=1家事、行全体がタップ領域
- 画面最上部に「最近記録した家事 上位3件」のクイックボタンを置く(直近30日の記録回数順)
- 家事をタップ → 記録モーダルを開く

**記録モーダル:**

| 項目 | 仕様 |
|---|---|
| 家事名 | 表示のみ |
| 実施者 | セグメントボタン3択: A / B / 二人で。デフォルト=現在の操作ユーザー |
| 日時 | `<input type="datetime-local">`。**デフォルト=現在時刻**。過去の記録漏れ入力に対応 |
| 保存 | logs に insert → トースト「記録しました」→ モーダルを閉じる |

保存は1タップ+確認1タップで完了する軽さを最優先にする。

### 4.2 草タブ(月表示・GitHub風コントリビューショングラフ)

- 月単位のカレンダーグリッド(7列、月曜始まり)。前月/次月ナビ付き
- **各日セルを上下2分割**: 上半分=ユーザーA、下半分=ユーザーB。それぞれの当日件数で濃淡4段階(0件=ほぼ透明、1 / 2-3 / 4件以上)。`both` は両方にカウント
- これがこのアプリの署名的ビジュアル。「どちらかだけ濃い日」「両方濃い日」が一目でわかる
- 日セルをタップ → その日の記録一覧をボトムシートで表示(時刻順)。各行に **編集・削除** ボタン
  - 編集: 記録モーダルを既存値入りで開き update
  - 削除: confirm 後 delete
- 月の下部に当月サマリ: A ○件 / B ○件(bothは両方に含む)、うち二人で ○件

### 4.3 週タブ(時間割ビュー)

- 「いつやっているか」を見るための画面。縦=時刻、横=曜日(月〜日)のタイムテーブル
- 表示時間帯は 5:00〜25:00(=翌1:00)、縦スクロール。深夜1時以降の記録は25時側に丸めて表示
- 各記録を実施者色の**チップ(家事名の短縮ラベル付き)**として done_at の位置にプロット。`both` は2色の斜めストライプ
- 同時間帯に重なる場合は横に並べる or 件数バッジでまとめ、タップで一覧表示
- 前週/次週ナビ、今週ボタン付き。週は月曜始まり

### 4.4 集計タブ

- 期間切り替え: 今週 / 今月(前期間へのナビ付き)
- 上段: A vs B の件数を横棒で比較(bothは両方にカウント)+ 「二人で」件数を別枠表示
- 下段: 家事別の件数テーブル(家事名 | A | B | 二人で | 計)。件数降順
- グラフライブラリは使わず、divの横棒で十分

## 5. UIデザイン指針

- モバイルファースト(想定幅 375〜430px)。PCで開いても中央 480px カラムで破綻しないこと
- タップターゲットは最小 44px
- 配色: 背景は白〜ごく薄いグレー、A=ティール系 `#2A9D8F`、B=アンバー系 `#E9A03B`(config で変更可能。この2色は色覚多様性でも区別しやすい組み合わせを維持すること)。both は2色の斜めストライプまたは半々塗りで統一
- フォント: システムフォント(`-apple-system, "Hiragino Sans", "Noto Sans JP", sans-serif`)。数字はタブラー数字指定
- 装飾は最小限。署名要素は草グラフの2人分割セルに集中させ、他は静かに
- iOSホーム画面追加を想定し、`viewport` メタ、`apple-mobile-web-app-capable`、テーマカラー、簡易 favicon を設定

## 6. 実装上の注意

- **タイムゾーン**: DBは timestamptz(UTC)で保存し、表示・集計はすべて端末ローカル(JST)で行う。日付境界の判定は `done_at` をローカル日付に変換してから行うこと(UTCのまま日付を切ると9時間ずれる)
- **週の定義**: 月曜始まり
- データ取得は画面ごとに必要範囲のみ(草タブ=当月±0、週タブ=当週、集計=当期間)。`done_at` の範囲クエリで絞る
- chores マスタは初回ロード時に全件取得しメモリ保持(`is_active = true` のみ表示)
- オフライン対応・リアルタイム同期は不要。画面切り替え時と保存後に再取得すれば十分
- 楽観的UI不要。保存失敗時はトーストでエラー表示し、入力値は保持する
- エラー文言は具体的に(例: 「保存できませんでした。電波状況を確認してもう一度お試しください」)

## 7. セキュリティに関する割り切り(利用者向け注記)

anon キーはフロントに埋め込むため公開情報になる。RLSを全許可にしているので、**URLとキーを知っていれば誰でも読み書きできる**。2人用の家事記録という性質上、実害は小さいと判断してMVPではこの構成とする。気になる場合の将来オプション: Supabase の匿名認証+共有パスフレーズ、またはメールマジックリンク認証。

## 8. セットアップ手順(README に含めること)

1. Supabase でプロジェクト作成(リージョン: Tokyo)
2. SQL Editor で §3 のテーブル定義・RLSポリシー・シードデータを実行
3. Project Settings → API から URL と anon キーを取得し `config.js` に記入
4. GitHub リポジトリを作成し4ファイルを push、Settings → Pages で `main` ブランチ公開
5. 2人ともそのURLをスマホのホーム画面に追加
